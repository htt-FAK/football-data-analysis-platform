## 二、首次部署（完整流程）

### 步骤 ① 传脚本 + 跑 deploy.sh（装环境）

**在本地电脑执行：**

```bash
# 1. 把部署脚本和 SQL 传到服务器 /root/
scp deploy.sh init_database.sql root@118.126.102.143:/root/

# 2. 登录服务器
ssh root@118.126.102.143

# 3. 执行一键部署（约 10-15 分钟，取决于网络）
cd /root
chmod +x deploy.sh
bash deploy.sh
```

`deploy.sh` 会自动完成：

| 步骤 | 内容 |
|------|------|
| 0 | 检查/启动 Hadoop |
| 1 | 验证 MySQL 连接（root1@118.126.102.143） |
| 2 | 安装并启动 Redis |
| 3 | 安装 Python + venv + 依赖 |
| 4 | 安装 Node.js 18 |
| 5 | 执行 init_database.sql 建 12 张表 |
| 6 | 初始化 HDFS 目录（21 个子目录） |
| 7 | 创建项目骨架 /opt/sports/project/ |
| 8 | 创建 React 前端项目 |
| 9 | 生成 .env / requirements.txt |
| 10 | 生成启动脚本 |
| 11 | 环境验证报告 |

### 步骤 ② 同步真实代码（关键！）

⚠️ `deploy.sh` 只创建了**空目录 + 空 `__init__.py`**，真正写好的代码要单独同步。

**在本地电脑执行：**

```bash
# 在项目根目录（含 backend/ 的那一层）执行
bash sync_code.sh
```

`sync_code.sh` 会用 rsync 把 `backend/app/` 全部代码同步到服务器，并做以下检查：
- 服务器连通性
- 关键文件是否到位（main.py / websocket.py / ingest_service.py）
- 自动复制 .env（没有则从 .env.example 创建）

> 💡 如果服务器 IP 或用户名变了，用环境变量覆盖：
> ```bash
> SERVER_USER=root SERVER_HOST=1.2.3.4 bash sync_code.sh
> ```

### 步骤 ③ 启动服务

```bash
ssh root@118.126.102.143

# 启动 Hadoop（如未运行）
bash /opt/sports/project/scripts/start_hadoop.sh

# 启动 Redis
bash /opt/sports/project/scripts/start_redis.sh

# 启动后端 API（含 WebSocket + APScheduler 自动起）
bash /opt/sports/project/scripts/start_backend.sh
# Swagger:  http://118.126.102.143:8000/docs
# WebSocket: ws://118.126.102.143:8000/ws

# 另开终端启动前端
bash /opt/sports/project/scripts/start_frontend.sh
# http://118.126.102.143:5173
```

### 步骤 ④ 验证

浏览器打开以下地址，确认服务正常：

| 地址 | 预期 |
|------|------|
| http://118.126.102.143:8000/docs | Swagger 接口列表（约 30 个接口） |
| http://118.126.102.143:5173 | React 前端（目前是 Vite 默认页） |

## 三、后续更新代码（迭代开发）

每次在本地改完代码后，只需两步：

# 2. 重启后端（让代码生效）

## 四、数据库表结构升级（v1.0 → v2）

如果服务器上之前部署过 **v1.0 的 10 表版本**，需要升级表结构。`init_database.sql` 已内置**幂等升级语句**，直接重新执行即可：

```bash
ssh root@118.126.102.143
mysql -h 118.126.102.143 -u root1 -p root1 < /root/init_database.sql
```

升级会自动补充：
- `players` 表 +11 个字段（门将 4 个 + 六边图评分 7 个）
- `team_stats` 表 +5 个字段（matches_played / wins / draws / losses / clean_sheets）
- 新建 `data_sources` / `crawl_logs` 表（如不存在）

升级语句用存储过程判断"列是否存在"，**多次执行不会报错**。

## 五、常用排障

### 后端启动失败

```bash
# 看实时日志
ssh root@118.126.102.143
source /opt/sports/venv/bin/activate
cd /opt/sports/project/backend
python -c "from app.main import app; print('OK')"
# 如果报错会直接显示，根据错误修复
```

### MySQL 连接失败

```bash
# 在服务器测试连接
mysql -h 118.126.102.143 -u root1 -p
# 密码见 .env: DB_PASSWORD=xiangmushu
```

### Redis 连接失败

```bash
systemctl status redis
systemctl restart redis
redis-cli ping  # 应返回 PONG
```

### HDFS 目录丢失

```bash
hdfs dfs -ls /sports/
# 没有则重新初始化（deploy.sh 步骤 6 的命令）
```
